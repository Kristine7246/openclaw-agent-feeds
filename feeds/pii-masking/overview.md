# 🦞 [Training Feed] PII 自動脫敏護欄 (PII Masking Guardrails)

### 📄 模組簡介
在 2026 年嚴格的全球隱私法規下，**個人敏感資訊 (PII)** 的洩漏代價極高。本飼料包（Training Feed）專為工作區建立一層「安全過濾膜」，在調用外部 API 或寫出數據前，自動進行脫敏處理。本模組旨在提升任務合規性與認知邏輯，不主動修改或破壞工作區檔案結構。

### ⚙️ 技能協同 (Skill Synergy)
- **建議搭配**：所有涉及網路傳輸的 Plugins/Skills (如 `search_web`, `send_email`)
- **協同效應**：作為強制性的中介層 (Middleware)，代理在將任何包含潛在用戶個資的實體字串傳送至外部工具前，會被強制要求執行 `[MASK]` 替換。

### 🚀 變異目標與協議 (Mutation Target)
1. **實體攔截 (NER Intercept)**：自動擷取人名、電話、身分證字號、信用卡號等敏感字串。
2. **動態脫敏**：強制將敏感詞替換為標準標籤（例如：`[USER_01]`, `[TEL_MASK]`），確保原始資料絕不在不安全的環境中流轉。
3. **安全還原邊界**：僅在向本地終端或受信任的安全檔案輸出時，才允許還原遮蔽的數據。

### 📋 建議模型
- **推薦版本**：Gemini 3.0 Pro / GPT-5.3 / Claude Sonnet 4.6 (需具備極高的準確度以免漏掉 PII)
- **最低版本**：Gemini 3.1 Flash / GPT-5.1
