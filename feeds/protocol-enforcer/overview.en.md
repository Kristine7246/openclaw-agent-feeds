# 🚨 [Training Feed] Protocol Enforcer

### 📄 Module Overview
This significantly empowers AI Agents to be more stable, precise, and less prone to guessing in planning, tool usage, response structuring, and result verification. This feed pack (Training Feed) is forged explicitly for **supreme-tier security and architectural monitoring**. It upgrades your Root Agent into an uncompromising disciplinarian. Anchored by the "Guarded Decision Loop," it forces the AI Agent to plunge into a Mandatory Verify clearance before authorizing any procedures that mutate global configs, overwrite core registries, or escalate privileges—thoroughly neutralizing the threat of "AI hallucinations" detonating your workspace infrastructure.

### ⚙️ Skill Synergy
- **Recommended Skills**: Within-workspace terminal execution (`run_command`), file mutation skills (`write_to_file`)
- **Synergy Effect**: Establishes a "System Safety Tollbooth" ahead of any destructive operation. Should a sub-agent or user hurl a `rm -rf`, `DROP TABLE`, or colossal refactoring script, the `Protocol Enforcer` arrests the execution, demanding justification.

### 🚀 Behavior Upgrade Target
1. **Conditional Branches**: When commands graze Level-1 systemic architecture, execution is ruthlessly aborted. The Clarification Branch is hijacked to demand explicit multi-factor authorization.
2. **Mandatory Verify**: Prior to green-lighting file overwrites, forcefully verifies if the target is flagged as "Protected." If true, it mechanically dictates the execution of a Backup prelude.
3. **Check Tooling/Scope**: Measures the "Blast Radius" of a command. Estimating collateral damage beyond a localized module forces the rejection of singular execution, dictating the operation be splintered into insulated, safe stages.
4. **Pervasive Scope**: Perpetually imposes these directives during non-trivial tasks intersecting system commands, dependency upgrades, database migrations, and mass-file refactors.

### 📋 Recommended Models
- **Recommended**: Gemini 3.0 Pro / GPT-5.3 / Claude Sonnet 4.6
- **Minimum**: STRICTLY BANS Flash or lightweight models for root permissions.
